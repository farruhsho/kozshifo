# KO'Z SHIFO — агент звонков (Android)

Маленькое приложение на **телефон ресепшена**. Слушает состояние телефона,
определяет каждый завершённый звонок (входящий / исходящий / пропущенный /
отклонённый), время ожидания до ответа, и отправляет это на backend, чтобы
**директор** видел в приложении почти в реальном времени, вовремя ли ресепшен
отвечает на звонки пациентов.

> Запись разговора не ведётся — только метрики (статус, время ответа,
> длительность). Полный контракт с backend: [`docs/INTEGRATIONS_CALL_AGENT.md`](../docs/INTEGRATIONS_CALL_AGENT.md).

## Что внутри

- `CallMonitorService` — foreground-сервис: слушает `TelephonyManager`
  (RINGING → OFFHOOK → IDLE) для измерения времени ожидания, читает завершённый
  звонок из `CallLog`, кладёт в очередь и раз в минуту отправляет пачкой +
  heartbeat. После перезагрузки восстанавливается через `BootReceiver`; на каждом
  тике сверяется с `CallLog`, чтобы не потерять звонки, пока приложение не работало.
- `UploadQueue` — стойкая SQLite-очередь, удаляет записи только после ответа
  сервера (переживает офлайн / разряд батареи). Идемпотентность по `CallLog._ID`.
- `ApiClient` — `HttpURLConnection`, без сторонних зависимостей.
- `MainActivity` — экран настройки (адрес сервера + ключ, разрешения, старт/стоп).

Зависимости минимальны: `androidx.core` + корутины. Никаких Retrofit/OkHttp/Room.

## Сборка

Требуется **JDK 17** и Android SDK (compileSdk 34).

**Вариант А — Android Studio:** `File → Open` → выберите папку `call-agent`,
дождитесь Gradle sync, затем `Run` (или `Build → Build APK(s)`).

**Вариант Б — командная строка:** в папке `call-agent` сначала создайте wrapper
(jar не закоммичен), потом соберите debug-APK:

```bash
gradle wrapper --gradle-version 8.7
./gradlew assembleDebug
# APK: app/build/outputs/apk/debug/app-debug.apk
```

`local.properties` с `sdk.dir=...` Android Studio создаёт сам; для CLI укажите
переменную `ANDROID_HOME`.

## Установка на телефон ресепшена (sideload)

Приложение распространяется как APK напрямую (не через Google Play — поэтому
разрешения `READ_CALL_LOG` / `READ_PHONE_STATE` доступны без ограничений Play).

1. Скопируйте `app-debug.apk` на телефон, установите (разрешите установку из
   неизвестных источников).
2. Откройте приложение. В приложении директора: **Звонки → Телефоны ресепшена →
   создать телефон** — скопируйте **адрес сервера** и **ключ**.
3. Вставьте их в агент → «Выдать разрешения» (телефон + журнал звонков, на
   Android 13+ ещё уведомления) → «Отключить экономию батареи» → «Сохранить и
   запустить».
4. Готово: статус показывает все ✓. Сделайте тестовый звонок — через ~минуту он
   появится у директора.

## Замечания

- На некоторых прошивках (Xiaomi/MIUI, Huawei, Oppo, …) нужно дополнительно
  разрешить «автозапуск» приложения и снять ограничения фоновой работы вручную —
  иначе система убивает сервис.
- При первом запуске история звонков не загружается (старт с текущего момента),
  чтобы не засорять журнал старыми звонками.
